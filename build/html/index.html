GET /feed/

The feed object contains the id, model_name, and created_at fields. The model_name tells you what model it is fetched from so you can use the correct endpoint to fetch the full object. This feed is peculiar in that it returns two different sets of feed, a default feed which contains personalized content and a support feed which comes when the default is exhausted, they have different paginator names and numbers so when ** **has_next is false for the default, you should check the first page for the support feed.

Unified feed

Status Codes:

  • 200 OK --
